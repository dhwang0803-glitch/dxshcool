🚀 설정 방법:

Step 1: 폴더 생성
$ mkdir -p database-design/.claude
$ mkdir -p rag-pipeline-design/.claude

Step 2: claude.md 복사
$ cp claude_md_database_design.md database-design/.claude/claude.md
$ cp claude_md_rag_pipeline.md rag-pipeline-design/.claude/claude.md

Step 3: 데이터 폴더 연결
$ ln -s ../prepared_data database-design/data/
$ ln -s ../rag_analysis rag-pipeline-design/data/

Step 4: Claude Code에서 각 프로젝트 열기
- database-design/ 폴더 열기 → Claude Code 시작
- rag-pipeline-design/ 폴더 열기 → Claude Code 시작

📝 두 .claude/claude.md 파일의 차이:

┌─ Database Design ─────────────────────┐
│ 목표: PostgreSQL 스키마 설계          │
│ 입력: 샘플 데이터 + 메타데이터        │
│ 출력: DDL 스크립트 + 설계 문서        │
│ 기간: 1-2주                           │
│ 우선순위: 1순위 (먼저 진행)           │
└───────────────────────────────────────┘

┌─ RAG Pipeline Design ─────────────────┐
│ 목표: 결측치 자동 채우기 파이프라인    │
│ 입력: 결측치 메타데이터 + 검색 전략    │
│ 출력: 검색 함수 + 파이프라인           │
│ 기간: 2-3주                           │
│ 우선순위: 2순위 (DB 완료 후)          │
└───────────────────────────────────────┘
