# GET /similar/{asset_id} — 유사 콘텐츠

## 기능 개요

기준 VOD와 유사한 콘텐츠 목록을 반환한다. 콘텐츠 기반 추천(VISUAL_SIMILARITY, CONTENT_BASED) 우선, 없으면 동일 장르 fallback.

## 기본 기능

**요청**:
```
GET /similar/{asset_id}?limit=10
```
- `asset_id`: `full_asset_id` (예: `cjc|M1234567LSGH00000001`)
- `limit`: 1~50, 기본 10

**처리 로직**:
1. `serving.vod_recommendation` 에서 `source_vod_id` 기준 조회
   - `recommendation_type IN ('VISUAL_SIMILARITY', 'CONTENT_BASED')`
   - TTL 필터: `expires_at IS NULL OR expires_at > NOW()`
2. 없으면 동일 장르 VOD로 fallback:
   ```sql
   WHERE genre = (SELECT genre FROM vod WHERE full_asset_id = $1)
     AND full_asset_id <> $1
   ```

**응답**:
```json
{
  "base_asset_id": "cjc|M1234...",
  "items": [
    {
      "asset_id": "cjc|M5678...",
      "title": "옥자",
      "genre": "드라마",
      "poster_url": "/posters/...",
      "score": 0.89,
      "rank": 1
    }
  ],
  "total": 10,
  "source": "vector_similarity"
}
```

- `source`: `"vector_similarity"` (추천 성공) | `"genre_fallback"` (장르 대체)

## 예외사항

| 에러 코드 | HTTP | 조건 | 메시지 |
|-----------|------|------|--------|
| `SIMILAR_NOT_FOUND` | 404 | 추천 + 장르 fallback 모두 결과 없음 | 유사한 콘텐츠를 찾을 수 없습니다 |

## 제약사항

- 인증 불필요
- `source_vod_id` 사용 (user_id_fk 아님 — 2026-03-20 버그 수정 완료)
- `serving.vod_recommendation` 미생성 시 자동 fallback (Exception 무시)
- TTL 필터 적용으로 만료된 추천 결과 제외

## 업스트림 의존성

| 테이블 | 컬럼 | 타입 | 용도 |
|--------|------|------|------|
| `serving.vod_recommendation` | `source_vod_id`, `vod_id_fk`, `rank`, `score`, `recommendation_type`, `expires_at` | 각종 | 콘텐츠 기반 추천 |
| `public.vod` | `full_asset_id`, `asset_nm`, `genre`, `poster_url` | VARCHAR/TEXT | 장르 fallback + 메타데이터 JOIN |
