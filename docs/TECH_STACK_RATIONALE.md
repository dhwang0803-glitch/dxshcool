# 기술 스택 선정 근거 (Q&A)

> 프레젠테이션 예상 질의응답 — "왜 이 기술을 선택했는가?"

---

## 1. 왜 PostgreSQL + pgvector? (vs Milvus, Pinecone)

**인프라 제약 + 운영 단순화**

- 서버: Oracle 무료 티어 VPC (1 core / 1GB RAM) → Milvus 최소 요구 8GB RAM, Pinecone은 유료 SaaS
- pgvector를 쓰면 **메타데이터 JOIN + 벡터 검색을 하나의 쿼리**로 처리 → 별도 벡터DB 운영 시 발생하는 데이터 동기화 문제 없음
- 16만 6천 건 규모에서 IVF_FLAT 인덱스로 충분한 성능 검증 완료

---

## 2. 왜 FastAPI? (vs Django, Flask)

**비동기 네이티브 + 자동 API 문서화**

- 추천 API는 pgvector 검색 → CF 결과 조회 → 리랭킹을 순차 호출 → asyncpg와 결합하여 DB I/O 대기를 비동기 처리
- Django: ORM이 강점이나, 직접 SQL 사용하는 구조에서는 오히려 오버헤드
- Flask: 비동기 지원이 네이티브가 아님
- Pydantic v2 스키마 검증 + Swagger 자동 생성 → 프론트엔드 팀과 협업 효율

---

## 3. 왜 CLIP? (vs ResNet, CNN 임베딩)

**멀티모달 통합 검색**

- CLIP은 영상 프레임과 텍스트를 **같은 512차원 공간**에 임베딩
- "로맨스 영화" 텍스트 쿼리로 영상 프레임 검색, 영상 프레임에서 음식/관광지 개념을 텍스트 매칭 가능
- ResNet: 이미지 분류에 강하지만, 텍스트-이미지 교차 검색 불가
- YOLO가 못 잡는 추상적 개념("벚꽃 봄 풍경", "전통시장 분위기")을 CLIP이 보완

---

## 4. 왜 YOLOv11? (vs v5, v8)

**소형 모델 기준 정확도 대비 추론 속도 최적**

- 한식 71종 커스텀 모델 학습
- 로컬 GPU: GTX 1650 Ti (4GB VRAM) → 모델 크기 제약이 큼
- v11s(소형)가 이 환경에서 실시간에 가까운 프레임 처리 가능

---

## 5. 왜 Ollama + Gemma? (vs ChatGPT API)

**비용 + 데이터 보안**

- 16만 건 VOD × 개인화 추천 문구 생성 → ChatGPT API 비용 수백만 원 단위
- Ollama 로컬/Colab 오픈소스 모델(Gemma3 27B) → **API 비용 0원**, VOD 메타데이터 외부 유출 없음
- gemma3:12b → 27b 업그레이드 품질 비교 완료: 27B가 복합 지시(길이 제한 + 금칙어 + 페르소나)를 안정적으로 처리

---

## 6. 왜 implicit(ALS)? (vs NCF, DeepFM)

**암시적 피드백 데이터 특성에 최적화**

- IPTV 시청 데이터: 평점 없음, 시청 여부(0/1) + 시청 시간만 존재하는 implicit feedback
- ALS: implicit feedback 특화, 16만 건 × 수천 유저 규모에서 CPU만으로 수 분 내 학습 완료
- DeepFM/NCF: GPU 필요, 이 규모에서 ALS 대비 유의미한 정확도 향상 없음

---

## 7. 왜 Whisper + EasyOCR 조합?

**인식 대상이 다름 — 4종 결합으로 광고 트리거 정확도 극대화**

| 모델 | 인식 대상 | 예시 |
|------|----------|------|
| YOLO | 시각 객체 (음식, 사물) | 화면 속 추어탕 그릇 |
| CLIP | 추상적 개념 | "벚꽃 봄 풍경", "전통시장 분위기" |
| Whisper | 음성 (나레이션, 대사) | "하동에서 녹차밭을 방문했습니다" |
| EasyOCR | 화면 텍스트 (자막, 간판) | "하동군 녹차연구소" 간판 |

4종 시그널을 결합하여 제철장터(음식) / 지자체 광고(관광지) 트리거 정확도를 높인 것이 핵심

---

## 8. 왜 Cloud Run? (vs EC2, 직접 서버)

**자동 스케일링 + 비용 최적화**

- 테스트 단계: 트래픽 없을 때 0 인스턴스 → 과금 없음
- EC2: 24시간 가동 시 비용 발생, VPC는 1 core라 API 서버 추가 운영 어려움
- Docker 이미지 빌드만으로 배포 자동화 → 소규모 팀에 적합

---
