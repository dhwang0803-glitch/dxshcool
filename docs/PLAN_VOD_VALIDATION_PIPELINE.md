# VOD 데이터 정합성 검증 파이프라인 계획

> 작성일: 2026-04-08
> 근거: Notion "추천시스템_데이터 정합성 점검 리포트"

## 배경

팀원 전수 점검 결과, IPTV 원천 데이터에서 다음 4가지 유형의 데이터 품질 문제가 발견됨.

| 유형 | 건수 | 예시 |
|------|------|------|
| 원본 삭제/재생 불가 | 8회차 | 런닝맨 597·600·601·624, 살림남 265·275, 공룡대탐험 1, 결혼작사이혼작곡3 1 |
| 동일 영상 중복 매핑 | 50+ 회차 | 뭉찬2(7그룹), 살림남(5그룹), 1박2일시즌4(7그룹) |
| 시즌 간 에피소드 혼입 | 20+ 회차 | 1박2일시즌4 내 시즌1 영상, 결혼작사이혼작곡3 내 시즌2/외부채널 |
| 날짜 이상치 | 다수 | 2021~2023년 사이에 2025~2026년 날짜 혼재 |

## 근본 원인

IPTV 제공사의 `full_asset_id → 실제 영상 파일` 매핑이 잘못됨. 우리 시스템이 이를 검증 없이 그대로 서빙 중.

## 해결 방향: YouTube를 정본(canonical) 영상 소스로 활용

### 현재 구조
```
full_asset_id → IPTV 영상 (매핑 오류 다수)
youtube_video_id → 트레일러만 (보조)
```

### 목표 구조
```
full_asset_id → IPTV 영상 (1차 소스)
                  ↓ 재생 실패 or 검증 실패 시
youtube_video_id → 공식 채널 풀영상 (fallback)
```

### 구현 계획

#### 1단계: 회차별 YouTube 풀영상 매칭 파이프라인
- VOD_Embedding 트레일러 크롤링 파이프라인을 확장
- `series_nm + 회차` 조합으로 공식 채널(KBS, SBS, MBC, tvN 등)에서 해당 에피소드 검색
- YouTube Data API v3 `search.list` 활용, 공식 채널 ID 필터
- 매칭 결과를 `youtube_video_id` 컬럼에 저장

#### 2단계: 중복 영상 자동 감지
- 같은 series_nm 내에서 `youtube_video_id`가 동일하면 중복 확정
- YouTube 영상 `duration`이 동일하면 의심 플래깅
- 감지 결과를 리포트로 출력 → 수동 검토 후 조치

#### 3단계: 시즌 혼입 교차 검증
- YouTube 매칭 결과의 영상 제목에서 시즌 정보 추출
- DB의 series_nm과 비교하여 불일치 시 자동 플래깅
- 예: DB `1박2일 시즌4 134회` ↔ YouTube `1박2일 시즌1 134회` → 시즌 혼입 감지

### 한계
- 모든 에피소드가 YouTube에 있진 않음 (특히 유료 VOD)
- 공식 채널만 사용 (불법 업로드 제외)
- YouTube API 일일 할당량 10,000 units 제한

## 임시 조치 (2026-04-08 적용)

발표 시 문제 VOD 재생 방지를 위해, 리포트에 명시된 문제 에피소드들을 유료(`PPV`)로 전환.
추후 YouTube 매칭 완료 시 원복 예정.
