# CF_Engine 추천 급락 지점 분석 리포트

- **작성일**: 2026-03-17
- **브랜치**: CF_Engine
- **모델**: ALS (factors=128, iterations=20, regularization=0.01, alpha=40)
- **분석 목적**: 유저별로 "의미있는 추천 구간"과 "리스트 채우기 구간"의 경계를 파악하여 최적 K 및 폴백 전략 수립

---

## 1. 분석 배경

ALS 모델은 유저에게 Top-10 추천을 제공하지만, 실제로 **모델이 확신을 갖고 추천하는 건 몇 개인지** 파악이 필요했다.
추천 점수가 급격히 떨어지는 지점을 기준으로 "진짜 추천"과 "리스트 채우기"를 구분한다.

- **샘플**: 500명 (시청 이력 5개 이상)
- **비교**: 필터 없음(27.9% 저품질 혼합) vs 품질 필터 ON(poster_url + vod_embedding 둘 다 있는 VOD만)

## 5. 핵심 인사이트

### 왜 1~2개밖에 확신 못 하나?

| 원인 | 내용 |
|------|------|
| 학습 데이터 부족 | watch_history가 **1개월치**(2023-01)만 존재 |
| 저품질 VOD 27.9% 혼입 | 추천 후보에 poster/embedding 없는 VOD가 섞여 점수 분산 |
| ALS 알고리즘 특성 | 가장 유사한 1~2개에 점수가 몰리는 구조적 한계 |

### 품질 필터의 효과

> 필터 ON 시 NDCG +13%, HitRate +9.1%로 **정확도는 향상**되지만,
> 급락 패턴 자체는 거의 변하지 않음 → 저품질 제거만으로는 구조적 한계 극복 불가

## 6. 권장 K 및 폴백 전략

### K 결정

| 기준 | 권장 K | 커버 유저 | 근거 |
|------|--------|---------|------|
| 확실한 추천만 | K=1~2 | 75% | 모델 확신 구간 |
| **현실적 서비스 기준** | **K=10** | ~98% | 다양성 + 폴백으로 보완 |

> **최종 권장: K=10 유지**

### 폴백 전략 (제안)

```
추천 생성 로직:
1. 필터 ON 모델로 Top-5 생성  (모델 확신 구간)
2. 6~10위: 저품질 VOD 중 전체 시청 수 상위로 채움  (인기 기반 폴백)
3. 항상 Top-10 보장
```

| 구간 | 전략 | 근거 |
|------|------|------|
| **1~5위** | 고품질 VOD (필터 ON) | 90% 유저의 의미있는 추천 구간 |
| **6~10위** | 저품질 VOD 인기 순 폴백 | 다양성 확보 + 과적합 방지 |

## 7. 다음 단계

| 우선순위 | 작업 | 기대 효과 |
|----------|------|----------|
| 높음 | `train.py`에 1~5위 필터 ON + 6~10위 폴백 로직 구현 | 서비스 품질 즉시 개선 |
| 높음 | 폴백 적용 후 성능 재평가 (현재 HitRate 0.4176과 비교) | 효과 검증 |
| 중간 | **Vector_Search HYBRID 연계** | 의미있는 추천 5개까지 확장 가능 |
| 낮음 | 시청 이력 데이터 추가 확보 (1개월 → 6개월+) | 장기적 ALS 성능 개선 |

> **HYBRID 연계가 핵심**: CF가 확신하는 1~2개 + Vector_Search 콘텐츠 유사도로 3~5위 보완 시
> 현재 75% 유저 기준 2개 → **5개까지 의미있는 추천 확장 가능**
