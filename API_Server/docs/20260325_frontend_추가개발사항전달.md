## 변경사항 1 — `/vod/{asset_id}` 응답에 `youtube_url` 추가

### 변경 내용

`GET /vod/{asset_id}` 응답에 `youtube_url` 필드가 추가되었습니다.

```json
{
  "asset_id": "...",
  "title": "...",
  "genre": "...",
  "category": "...",
  "director": "...",
  "cast_lead": "...",
  "cast_guest": "...",
  "summary": "...",
  "rating": "...",
  "release_year": 2023,
  "poster_url": "https://...",
  "is_free": false,
  "youtube_url": "https://www.youtube.com/embed/abc1234XYZ"
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| `youtube_url` | `string \| null` | YouTube iframe embed URL. DB에 `youtube_video_id`가 없으면 `null` 반환 |

### Frontend 처리 방침

- `youtube_url`이 `null`이 아닌 경우 → 트레일러 재생 버튼/섹션 노출
- `youtube_url`이 `null`인 경우 → 트레일러 섹션 숨김 처리

## 변경사항 2 — YouTube iframe 트레일러 재생 + 진행률 연동

### 배경

트레일러 영상은 YouTube iframe embed 방식으로 제공됩니다. 일반 `<iframe src="...">` 태그로는 재생 위치를 읽을 수 없으므로 **YouTube IFrame Player API**(`YT.Player`)를 사용해야 합니다.

### 구현 방법

**1단계 — YouTube IFrame Player API 스크립트 로드**

```html
<script src="https://www.youtube.com/iframe_api"></script>
```

**2단계 — YT.Player 초기화 및 heartbeat 연결**

```js
let player;

function onYouTubeIframeAPIReady() {
  player = new YT.Player('yt-player', {
    videoId: extractVideoId(youtube_url), // embed URL에서 ID 추출
    playerVars: { autoplay: 0 },
    events: {
      onReady: startHeartbeat,
    },
  });
}

function extractVideoId(embedUrl) {
  // "https://www.youtube.com/embed/abc1234XYZ" → "abc1234XYZ"
  return embedUrl.split('/embed/')[1];
}

function startHeartbeat(event) {
  setInterval(() => {
    const current  = player.getCurrentTime(); // 현재 재생 위치 (초)
    const duration = player.getDuration();    // 전체 길이 (초)
    if (!duration) return;

    const completionRate = Math.min(100, Math.round((current / duration) * 100));

    // 기존 progress heartbeat 엔드포인트 재사용
    fetch(`/series/${seriesNm}/episodes/${assetNm}/progress`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${jwtToken}`,
      },
      body: JSON.stringify({ completion_rate: completionRate }),
    });
  }, 30000); // 30초마다
}
```

**3단계 — 진입 시 이전 진행률 복원**

```js
// 시리즈 상세 페이지 진입 시
const res  = await fetch(`/series/${seriesNm}/progress`, { headers: { Authorization: ... } });
const data = await res.json();
// data.episodes[assetNm].completion_rate → progress bar 초기값
```

### 진행률 흐름 요약

```
시리즈 상세 페이지 진입
  → GET /series/{series_nm}/progress
  → 에피소드별 completion_rate → progress bar 초기 렌더링

YouTube iframe 재생 중 (30초마다)
  → YT.Player.getCurrentTime() / getDuration()
  → completion_rate 계산
  → POST /series/{series_nm}/episodes/{asset_nm}/progress
  → 인메모리 버퍼 → 60초마다 DB UPSERT
  → UI progress bar는 로컬 state로 즉시 업데이트 (API 재조회 불필요)
```

### Progress bar 표기 기준

| 상태 | completion_rate | 표기 |
|------|----------------|------|
| 미시청 | 0 또는 없음 | progress bar 미표시 |
| 시청 중 | 1 ~ 99 | progress bar 표시 (비율 반영) |
| 시청 완료 | 100 | "시청 완료" 배지 또는 full bar |

## 주의사항

- `youtube_url`이 있더라도 YouTube 정책상 일부 영상은 embed 차단될 수 있습니다. `onError` 이벤트에서 플레이어 영역을 숨기는 fallback 처리를 권장합니다.
- heartbeat는 로그인 상태(JWT 보유)일 때만 전송합니다. 비로그인 사용자는 진행률 기록하지 않습니다.
- `YT.Player.getDuration()`은 영상이 완전히 로드된 후(`onReady` 이후)에만 정확한 값을 반환합니다.
