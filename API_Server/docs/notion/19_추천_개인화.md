# GET /recommend/{user_id} — 개인화 추천

## 기능 개요

CF_Engine/Vector_Search 결과 기반 개인화 추천 목록을 반환한다. 추천 데이터 없으면 인기 콘텐츠 fallback.

## 기본 기능

**요청**:
```
GET /recommend/{user_id}?limit=10
Authorization: Bearer {token}
```
- `limit`: 1~50, 기본 10

**처리 로직**:
1. `serving.vod_recommendation` 에서 `user_id_fk` 기준 조회 + TTL 필터 (`expires_at` 확인)
2. 없으면 `serving.mv_vod_watch_stats`에서 인기순 fallback
3. 둘 다 없으면 빈 배열

**응답**:
```json
{
  "user_id": "a1b2c3...",
  "items": [
    {
      "asset_id": "cjc|M1234...",
      "title": "기생충",
      "genre": "드라마",
      "poster_url": "/posters/...",
      "score": 0.95,
      "rank": 1,
      "recommendation_type": "COLLABORATIVE"
    }
  ],
  "total": 10,
  "source": "personalized"
}
```

- `source`: `"personalized"` (추천 성공) | `"popular_fallback"` (인기 대체)

## 예외사항

- 추천/인기 모두 없으면 빈 배열 (에러 아님)
- `serving` 스키마 미생성 시 자동 fallback (Exception 무시)

## 제약사항

- 인증 필수
- TTL 필터: `expires_at IS NULL OR expires_at > NOW()` (만료된 추천 제외)
- tag_recommendation 기반 패턴 그룹핑 구현 예정 (DDL 반영 후)
  - 현재: flat list 반환
  - 목표: 패턴 5개 x VOD 10개 그룹핑 구조

## 업스트림 의존성

| 테이블 | 컬럼 | 타입 | 용도 |
|--------|------|------|------|
| `serving.vod_recommendation` | `user_id_fk`, `vod_id_fk`, `rank`, `score`, `recommendation_type`, `expires_at` | 각종 | 개인화 추천 (user_id_fk 기준) |
| `serving.mv_vod_watch_stats` | `vod_id_fk`, `total_watch_count` | VARCHAR/INT | 인기순 fallback |
| `public.vod` | `asset_nm`, `genre`, `poster_url` | VARCHAR/TEXT | 메타데이터 JOIN |
