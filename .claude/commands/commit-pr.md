# /commit-pr

현재 브랜치에서 작업한 내용만 commit하고 PR을 생성한다.
**다른 브랜치 파일은 절대 스테이징하지 않는다.**

## 실행 절차

### 1. 현재 브랜치 및 변경 파일 확인
```bash
git branch --show-current
git status
git diff --stat
```

현재 브랜치명을 파악하고, 변경된 파일이 **현재 브랜치 폴더** 내에 있는지 확인한다.
다른 브랜치 폴더(예: VOD_Embedding/, RAG/ 등)의 파일은 절대 스테이징하지 않는다.

### 2. 보안 점검 (커밋 전 필수)
변경된 파일에 대해 아래 항목을 반드시 확인한다:

| 점검 항목 | 기준 |
|-----------|------|
| 하드코딩된 자격증명 | API 키, 비밀번호, 토큰 하드코딩 없어야 함 |
| os.getenv() 기본값 | 실제 IP, DB명, 사용자명 기본값 없어야 함 |
| .env 파일 포함 여부 | .gitignore에 .env 있는지 확인 |
| data/ 포함 여부 | .gitignore에 data/ 있는지 확인 |

### 3. 현재 브랜치 파일만 스테이징 및 커밋
```bash
git add {현재 브랜치 폴더}/
git commit -m "..."
```

### 4. push 전 pull
```bash
git pull origin {현재 브랜치}
git push origin {현재 브랜치}
```

### 5. 이전 PR 내용 확인
```bash
gh pr list --head {현재 브랜치} --state all --limit 1
gh pr view {PR번호} --json body
```

이전 PR이 있으면 body를 읽어 내용을 파악한다.
새 PR body 작성 시:
- 이전 PR과 **중복되는 항목**은 최신 내용으로 덮어써서 반영한다.
- 이번에 **새로 추가된 항목**은 해당 섹션에 추가한다.

### 6. 새 PR 생성
`gh pr create` 로 새 PR을 생성하며, body에 아래 세 섹션을 **반드시** 포함한다.

---

## PR Body 필수 포함 내용

### 변경사항 요약
변경된 파일별로 무엇을 왜 변경했는지 표 형식으로 기재.

### 변경 시 사후 영향 평가
| 영향 범위 | 내용 |
|-----------|------|
| 관련 테이블 | 읽기/쓰기 여부, 기존 데이터 영향 |
| 다른 브랜치 | 영향 없음 확인 |
| DB 부하 | 쿼리 실행 빈도 및 부하 수준 |

### 보안 평가
| 점검 항목 | 결과 |
|-----------|------|
| 하드코딩된 자격증명 | ✅/❌ |
| .env 직접 노출 | ✅/❌ |
| os.getenv() 기본값 인프라 노출 | ✅/❌ |
| .gitignore 확인 | ✅/❌ |
| DB 접속 방식 | ✅/❌ |

---

## 주의사항
- untracked 파일 중 다른 브랜치 관련 파일(VOD_Embedding/, RAG/ 등)은 스테이징 금지
- `.env`, `data/`, `*.parquet` 는 커밋 금지
- PR base branch는 항상 `main`
