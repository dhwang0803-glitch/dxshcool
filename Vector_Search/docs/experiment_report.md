# Vector Search 실험 리포트

**작성일**: 2026-03-17
**브랜치**: `Vector_Search`
**담당**: VOD 콘텐츠 기반 추천 엔진

---

## 1. 개요

VOD 메타데이터 임베딩(SBERT 384차원)과 CLIP 영상 임베딩(512차원)을 앙상블하여 콘텐츠 기반 유사 VOD 추천 결과를 생성하는 파이프라인을 구현하고, 설계 파라미터의 근거를 실험을 통해 검증한다.

## 2. 데이터 통계

| 항목 | 건수 | 비고 |
|------|------|------|
| 전체 VOD | 166,159건 | `vod` 테이블 |
| 메타데이터 임베딩 | 166,159건 | `vod_meta_embedding` |
| CLIP 임베딩 | 146,390건 | `vod_embedding` (clip-ViT-B-32) |
| **최종 추천 대상** | **119,730건** | poster_url + CLIP 임베딩 모두 보유 (72.1%) |
| 제외된 VOD | 46,429건 | 저품질 기준 미충족 (27.9%) |

### 저품질 VOD 제외 기준 및 근거

| 제외 기준 | 제외 이유 | 근거 |
|-----------|-----------|------|
| `poster_url` 없는 VOD | 화면에 노출할 포스터가 없어 추천해도 빈 카드로 표시됨 → 사용자 경험 저하 | UI/UX 관점에서 포스터 없는 VOD를 추천 목록에 노출하는 것은 의미 없음 |
| CLIP 임베딩 없는 VOD | 영상 기반 유사도 계산 불가 → 앙상블의 한 축이 완전히 빠지는 불완전한 추천 결과 생성 | 두 임베딩 모두 있는 VOD만 일관된 품질의 추천 가능 |

## 3. top_n=20 설정 근거

| 관점 | 근거 |
|------|------|
| **업계 표준** | Netflix, Amazon Prime 등 주요 스트리밍 서비스의 추천 리스트는 10~20개가 표준. 20개 이상은 사용자가 스크롤하지 않아 노출 효과 없음 |
| **API 유연성** | DB에 20개 저장 후 API_Server에서 `LIMIT`으로 실제 노출 수(5, 10개 등)를 별도 제어 가능 → 저장 단계에서 많이 확보할수록 서비스 레이어 유연성 증가 |
| **임계값 여유분** | 향후 score 임계값 필터링 적용 시 일부 결과가 제거될 수 있으므로 여유 있게 저장 필요. 본 실험에서 score < 0.6인 결과가 0.02%로 매우 적어 20개 저장이 충분함을 확인 |
| **MAP@k 관점** | 추천 평가 지표 MAP@k에서 k를 크게 설정할수록 다양한 추천 성능을 측정 가능. top_n=20은 MAP@5~20을 모두 측정할 수 있는 범위 |

## 4. 앙상블 설계

### 공식

```
final_score = α × clip_score + (1 - α) × content_score
```

### 두 임베딩의 역할

| 임베딩 | 차원 | 의미 | 특징 |
|--------|------|------|------|
| SBERT (content_score) | 384차원 | 장르/감독/배우/줄거리 등 텍스트 메타데이터 유사도 | 100% 커버리지, 구조화된 정보 |
| CLIP (clip_score) | 512차원 | 트레일러 영상의 시각적 분위기/톤 유사도 | 72.1% 커버리지, 비구조화 정보 |

### 특수 케이스 처리 및 근거

| 케이스 | 처리 방식 | 근거 |
|--------|-----------|------|
| CLIP 임베딩 없는 VOD | α = 0 → content_score 100% | CLIP 값이 없으면 앙상블 불가. 0으로 처리 시 content_score가 희석되는 문제 발생하므로 α를 아예 0으로 설정 |
| clip_score = 1.0 (시즌물) | α = 0 → content_score 100% | 동일 시리즈의 시즌물은 동일한 트레일러를 공유하는 경우가 있어 clip_score=1.0이 나타남. 이 경우 CLIP 유사도는 "동일 영상"을 의미할 뿐 실제 콘텐츠 유사성과 무관하므로 제외 |

## 5. alpha 최적화 실험

### 실험 설계

| 항목 | 내용 | 이유 |
|------|------|------|
| 샘플 수 | 500건 | 전체 119,730건 대비 통계적으로 유의미한 크기이면서 빠른 실험 가능 |
| 평가 지표 | Genre Precision@k | 사용자 행동 데이터 없는 현 단계에서 가장 객관적인 proxy 지표. 추천된 VOD가 기준 VOD와 장르가 일치하는 비율 측정 |
| 비교 alpha | 0.3, 0.4, 0.5 | CLIP과 content의 비율을 3:7, 4:6, 5:5로 나눠 메타데이터 우위 여부 검증 |

### 실험 결과

| alpha | clip:content 비율 | 장르 P@5 | 장르 P@10 | 장르 P@20 | 콘텐츠유형 P@10 | 선택 |
|-------|-----------------|---------|---------|---------|--------------|------|
| 0.3 | 3:7 | 62.1% | 51.1% | 41.5% | 53.5% | |
| **0.4** | **4:6** | **93.0%** | **91.2%** | **88.2%** | **95.9%** | ✅ |
| 0.5 | 5:5 | 59.4% | 47.8% | 38.1% | 50.8% | |

### alpha=0.4 선택 근거

| 근거 | 설명 |
|------|------|
| **실험 결과** | alpha=0.4가 모든 k(5, 10, 20)에서 압도적으로 높은 Precision 달성. alpha=0.3 대비 장르 P@10 기준 +40.1%p |
| **커버리지 비대칭** | CLIP 커버리지 72.1% vs 메타데이터 100% → 더 완전한 메타데이터에 가중치 우위 부여가 타당 (content 60%) |
| **데이터 신뢰도** | SBERT는 구조화된 한국어 메타데이터 기반으로 학습된 반면, CLIP은 영상 일반 모델로 한국 IPTV 도메인에 특화되지 않음 |

## 6. 추천 결과 score 분포 및 임계값 결정

### score 분포

| score 구간 | 건수 | 비율 | 품질 판단 |
|------------|------|------|----------|
| 0.9 ~ 1.0 | 1,974,914건 | 82.5% | 매우 높은 유사도 |
| 0.8 ~ 0.9 | 224,442건 | 9.4% | 높은 유사도 |
| 0.7 ~ 0.8 | 172,517건 | 7.2% | 보통 유사도 |
| 0.6 ~ 0.7 | 21,853건 | 0.9% | 낮은 유사도 |
| ~ 0.6 | 480건 | 0.0% | 저품질 |

| 통계 지표 | 값 |
|----------|-----|
| 평균 score | 0.958 |
| 중앙값 | 0.999 |
| 최솟값 | 0.523 |
| rank 20위 평균 | 0.942 |

### 임계값 결정: 별도 필터 불필요

score < 0.6인 결과는 **480건(0.02%)** 으로 사실상 무시할 수준. 임계값을 설정해 제거하더라도 전체 추천 품질에 미치는 영향이 미미하므로 별도 임계값 필터를 적용하지 않는다. 평균 score 0.958은 전반적으로 높은 추천 품질을 보장함을 의미한다.

## 8. 평가 지표 한계 및 향후 계획

### 현재 평가의 한계

| 한계 | 설명 |
|------|------|
| Proxy 평가의 순환성 | SBERT 임베딩이 장르/메타데이터 기반이므로, 장르 일치율로 평가하면 "메타로 만든 추천을 메타로 검증"하는 순환 논리 발생 |
| 사용자 만족도 미반영 | 장르가 같아도 실제로 보고 싶지 않은 VOD가 추천될 수 있음 |
| 정량적 정확도 한계 | 실제 클릭률(CTR), 시청 완료율 등 사용자 행동 데이터 없이는 진짜 정확도 측정 불가 |

### 향후 계획

| 단계 | 평가 방법 | 필요 데이터 | 기대 효과 |
|------|-----------|------------|----------|
| 서비스 오픈 후 | NDCG@k, MAP@k | 실제 CTR | 순서 고려한 정량 평가 |
| A/B 테스트 | alpha 재검증 (0.3~0.5 범위) | 사용자 행동 데이터 | 최적 alpha 데이터 기반 확정 |
| 시청 데이터 축적 후 | relevance = 시청 완료율 기반 NDCG | 시청 이력 | 만족도까지 반영한 고도화 평가 |
