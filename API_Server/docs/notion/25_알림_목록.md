# GET /user/me/notifications — 알림 목록

## 기능 개요

유저의 알림 목록을 최신순으로 조회한다. 미읽음 건수도 함께 반환.

## 기본 기능

**요청**:
```
GET /user/me/notifications
Authorization: Bearer {token}
```

**처리 로직**:
1. `notifications` 테이블에서 `user_id_fk` 기준 조회 (`created_at DESC`)
2. 미읽음 건수 계산 (`read = FALSE` 카운트)

**응답**:
```json
{
  "items": [
    {
      "id": 1,
      "type": "reservation",
      "title": "제철장터",
      "message": "채널 25번에서 제철장터이(가) 곧 시작됩니다",
      "image_url": null,
      "read": false,
      "created_at": "2026-03-25T13:55:00Z"
    }
  ],
  "total": 1,
  "unread_count": 1
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | int | 알림 고유 ID |
| `type` | string | 알림 유형 (`reservation` 등) |
| `title` | string | 알림 제목 |
| `message` | string | 알림 본문 |
| `image_url` | string? | 이미지 URL (선택) |
| `read` | bool | 읽음 여부 |
| `created_at` | datetime | 생성 시각 |
| `unread_count` | int | 전체 미읽음 건수 |

## 예외사항

- 알림 없으면 `items: []`, `total: 0`, `unread_count: 0` (에러 아님)

## 제약사항

- 인증 필수
- 전체 알림 조회 (읽음/미읽음 구분 없이)

## 업스트림 의존성

| 테이블 | 컬럼 | 타입 | 용도 |
|--------|------|------|------|
| `public.notifications` | `notification_id`, `user_id_fk`, `type`, `title`, `message`, `image_url`, `read`, `created_at` | 각종 | 알림 전체 조회 |
