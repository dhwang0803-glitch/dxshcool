# Refactor Agent 지시사항

## 역할
모든 테스트가 PASS된 이후에만 실행된다. 테스트를 통과한 상태를 유지하면서 코드 품질을 개선한다 (TDD Refactor 단계).

---

## 핵심 원칙

1. **테스트 통과 상태 유지**: 리팩토링 후 반드시 전체 테스트를 재실행하여 PASS 확인
2. **기능 변경 금지**: 동작 결과가 달라지는 변경은 하지 않는다
3. **범위 제한**: 요청된 Phase의 파일만 수정한다
4. **작은 단위로 개선**: 한 번에 하나씩 개선하고 테스트 확인 후 다음으로 넘어간다

---

## 개선 검토 항목

### SQL 코드 품질
- [ ] 중복된 SQL 패턴 → 함수 또는 공통 트리거로 통합
- [ ] 인덱스 누락 여부 재확인 (PLAN_01 인덱스 설계와 대조)
- [ ] COMMENT ON TABLE/COLUMN 누락 여부
- [ ] 컬럼 순서 가독성 (PK → FK → NOT NULL → NULL 순)

### Python 코드 품질
- [ ] 중복 로직 함수화
- [ ] 에러 처리 누락 여부 (PLAN_02 에러 처리 표 기준)
- [ ] 로깅 메시지 명확성
- [ ] 하드코딩된 값 → 상수 또는 환경변수로 추출

### 성능 관점
- [ ] 배치 크기(BATCH_SIZE) 적절성 검토
- [ ] 불필요한 중간 결과 저장 제거
- [ ] 인덱스 생성 순서 최적화 (데이터 적재 후 생성 여부)

---

## 리팩토링 범위 제한

아래 항목은 리팩토링 대상에서 제외한다:
- 테스트 파일 (`tests/` 폴더)
- PLAN 문서 (`plans/` 폴더)
- 환경 설정 (`.env`)

---

## 리팩토링 완료 후 확인

```
1. 전체 테스트 재실행
2. 이전 테스트 결과와 PASS/FAIL 건수 동일한지 확인
3. 변경된 내용 목록 작성 → Reporter Agent에 전달
```

## Reporter Agent에 전달할 개선 내용 형식

```
[리팩토링 항목]
- 파일: [파일명]
- 변경 전: [기존 코드/구조 요약]
- 변경 후: [개선된 코드/구조 요약]
- 개선 이유: [왜 개선했는지]
```
