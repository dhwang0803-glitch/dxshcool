## 1. 목적 — 왜 이걸 하는가

IPTV/케이블 VOD 예능 프로그램을 시청하는 중에, **화면에 나오는 음식이나 장면에 맞는 쇼핑 광고를 팝업으로 띄운다.**

예를 들어:
- 출연자가 **추어전**을 먹고 있다 → "남원 추어탕 밀키트" 팝업
- 출연자가 "이 **한우** 진짜 좋다"라고 말한다 → "횡성 한우 선물세트" 팝업
- **캠핑** 장면이 나온다 → "캠핑용 전기그릴" 팝업
- **바닷가 해변** 풍경이 나온다 → "제주 여행 패키지" 팝업

기존 TV 광고는 시간대별 일괄 편성이지만, 이 시스템은 **영상 내용을 실시간으로 분석하여 맥락에 맞는 광고를 자동 매칭**한다.

## 2. 어떤 광고를 띄우는가

### 광고 소스 2종

**1. 홈쇼핑 상품**
매일 자정 홈쇼핑 사이트(CJ온스타일 등)와 EPG 편성표를 크롤링하여 당일 방송 상품을 수집한다.

**2. 로컬 마켓 상품 (사용자 지역 기반)**
시청자의 거주 지역에 맞는 지역 특산물/로컬 마켓 상품을 추천한다.
- 시청자가 **예산** 거주 → "예산 사과", "예산 쌀" 광고
- 시청자가 **영광** 거주 → "영광 굴비", "법성포 젓갈" 광고
- 시청자가 **횡성** 거주 → "횡성 한우", "횡성 더덕" 광고

같은 VOD에서 같은 음식이 탐지되어도, **시청자 위치에 따라 다른 광고**가 노출된다.

```
[같은 VOD, 같은 장면: 소고기 구이]
  서울 시청자  →  "마장동 한우 직배송"
  횡성 시청자  →  "횡성 한우 로컬마켓"
  제주 시청자  →  "제주 흑우 농가 직송"
```

### 광고 카테고리

| 카테고리 | 트리거 예시 | 홈쇼핑 광고 예시 | 로컬 마켓 광고 예시 |
|---------|-----------|-------------|----------------|
| **한식/향토음식** | 추어전, 물회, 갈비 화면 | 남원 추어탕 밀키트 | 지역 한식당 밀키트 |
| **지방특산물** | "굴비" 발화, 생선구이 화면 | 영광 굴비 선물세트 | 근처 수산시장 굴비 |
| **과일/채소** | 사과, 감귤 화면 | 제주 감귤 5kg | 예산 사과 농가 직송 |
| **여행지** | 해변/리조트/제주 풍경 | 제주 여행 패키지 | 지역 펜션/숙소 |
| **주방가전** | 에어프라이어/그릴 사용 장면 | 에어프라이어 할인 | 지역 가전매장 |
| **건강식품** | "홍삼" 발화, 영양제 화면 | 홍삼 진액 | 풍기 인삼 로컬마켓 |
| **뷰티/패션** | 화장대, 옷 매장 장면 | 스킨케어 세트 | — |
| **운동/레저** | 골프장, 헬스장 장면 | 골프용품 | 지역 골프장 이용권 |

## 3. 어떻게 동작하는가

### 전체 흐름 요약

```
[1회 사전 처리]
VOD 트레일러 영상
  → 눈으로 본다 (YOLO): "이 프레임에 추어전이 있다" (bbox + 메뉴명)
  → 눈으로 본다 (CLIP): "이 장면은 캠핑이다" (장면 맥락)
  → 귀로 듣는다 (STT):  "출연자가 한우를 언급했다" (키워드)
  → 3종 결과를 DB에 저장

[매일 자정]
홈쇼핑 상품 크롤링 + 로컬 마켓 상품 DB 갱신
  → 탐지 결과와 매칭 → 광고 테이블 갱신

[시청 중 실시간]
영상 재생 → 현재 시간 + 시청자 위치 기준 광고 조회 → 팝업 표시
  → 홈쇼핑 광고 + 로컬 마켓 광고 동시 노출 가능
```

---

### Phase A: 사전 배치 처리 — "영상을 미리 분석한다"

VOD 트레일러를 로컬에서 3가지 방식으로 분석한다. 1회만 수행하면 되고, 분석 후 영상 파일은 삭제 가능.

#### 분석 1 — YOLO: "화면에 뭐가 보이는가"

```
영상 프레임 (1초에 1장 추출)
    → YOLO 모델이 음식을 찾는다
    → "이 프레임의 (x=120, y=80) 위치에 추어전이 있다, 신뢰도 0.87"
```

- **기본 모델**: COCO 80종 (사람, 자동차, 포크, 나이프 등 일반 물체)
- **파인튜닝 모델**: AI Hub 한식 761종 메뉴명 (추어전, 물회, 맥앤치즈, 동파육 등)
- 바운딩박스(bbox)로 화면 어디에 있는지 정확히 특정

#### 분석 2 — CLIP: "어떤 장면인가"

```
영상 프레임 전체 이미지
    → "바닷가 해변 바다" 텍스트와 유사도 0.31 → 여행지 장면
    → "전통시장 노점 길거리 음식" 유사도 0.28 → 지방특산물 장면
```

- 172개 한국어 쿼리와 이미지 유사도를 계산
- YOLO가 **물체**를 찾는다면, CLIP은 **장면 분위기**를 파악
- 음식 메뉴 구분은 못 함 (한국어 판별력 한계) → 장면 맥락에만 사용
- context_filter로 오탐 차단 (수족관, 장식 음식, 스튜디오 등)

#### 분석 3 — STT: "뭐라고 말하는가"

```
영상 오디오
    → Whisper 음성인식 → "이 영광 굴비가 정말 맛있네요"
    → 키워드 매칭 → "굴비" 히트 → 지방특산물 카테고리
    → ad_hints: ["영광 굴비 선물세트", "법성포 굴비"]
```

- 출연자가 직접 말하는 것을 인식 → 가장 정확한 구체적 정보
- 156개 키워드 (한우, 굴비, 대게, 제주, 에어프라이어 등)
- 말 안 하면 못 잡음 → YOLO가 시각적으로 보완

#### 3가지의 역할 분담

| | YOLO | CLIP | STT |
|---|---|---|---|
| **뭘 잡는가** | 구체적 메뉴명 (추어전, 물회) | 장면 분위기 (캠핑, 바닷가) | 발화 키워드 (한우, 굴비) |
| **강점** | 위치(bbox) 특정 가능 | 음식 외 장면도 파악 | 가장 구체적 |
| **약점** | 장면 맥락 모름 | 메뉴 구분 불가 | 말 안 하면 못 잡음 |
| **우선순위** | ★★★ | ★★ | ★★★ |

---

### Phase B: 매일 자정 크론 — "오늘의 광고 상품을 매칭한다"

```
홈쇼핑 크롤링 → 오늘 방송 상품 수집
  + 탐지 결과 (YOLO: "추어전", STT: "굴비", CLIP: "캠핑")
  → 매칭 엔진: 추어전 → "남원 추어탕 밀키트" 상품 연결
  → serving.shopping_ad 테이블에 저장
    (vod_id, 시작시간, 끝시간, 상품명, 가격, 구매링크, 이미지)
```

- 상품 정보를 비정규화하여 저장 → API가 JOIN 없이 바로 응답 가능
- TTL 30일 — 오래된 광고 자동 만료

---

### Phase C: 실시간 서빙 — "시청자에게 팝업을 띄운다"

```
시청자가 VOD 재생 중
  → 0.5초마다 현재 재생 시간 전송
  → API: "이 VOD의 120초 지점에 광고 있나?"
  → DB: "네, 추어전 밀키트 광고요"
  → UI: 팝업 표시 (상품 이미지, 이름, 가격, 구매 링크)
```

```sql
SELECT * FROM serving.shopping_ad
WHERE vod_id_fk = '해당VOD'
  AND ts_start <= 120.0 AND ts_end >= 120.0
  AND expires_at > NOW();
```

## 4. 구체적 시나리오

### 시나리오 1: 먹방 예능에서 추어전을 먹는 장면

```
[영상 120초 지점]
  화면: 출연자가 추어전을 먹고 있음
  음성: "와~ 이 추어전 진짜 바삭하다"

  YOLO → label: "추어전", confidence: 0.85, bbox: [200, 150, 450, 380]
  STT  → keyword: "추어전", ad_category: "지방특산물"
  CLIP → concept: "전통시장 노점 길거리 음식", ad_category: "장소"

  Shopping_Ad 매칭:
    YOLO "추어전" + STT "추어전" → 이중 확인 → score 높음
    → 홈쇼핑 상품 "남원 추어탕 밀키트 29,900원" 매칭
    → serving.shopping_ad에 저장

  시청자 VOD 재생 시 (시청자 위치: 예산):
    120초 도달 → API 조회 → 팝업:
    ┌─────────────────────────────────┐
    │ 🛒 남원 추어탕 밀키트 (홈쇼핑)   │
    │    29,900원  [구매하기]          │
    ├─────────────────────────────────┤
    │ 📍 예산 로컬마켓 추천            │
    │    예산 어죽 맛집  [자세히보기]   │
    └─────────────────────────────────┘
```

### 시나리오 2: 여행 예능에서 해변 장면

```
[영상 45초 지점]
  화면: 제주 바닷가 풍경
  음성: "제주도 바다가 정말 예쁘죠"

  YOLO → 음식 없음 (탐지 없음)
  STT  → keyword: "제주", ad_category: "여행상품"
  CLIP → concept: "모래사장 파란 바다", ad_category: "여행지"

  Shopping_Ad 매칭:
    STT "제주" + CLIP "여행지" → score 중간
    → 홈쇼핑 "제주 3박4일 패키지 399,000원" 매칭

  시청자 VOD 재생 시:
    45초 도달 → 팝업: "제주 여행 패키지"
```

### 시나리오 3: 캠핑 예능

```
[영상 80초 지점]
  화면: 야외 캠핑, 전기그릴에 고기 굽는 중
  음성: (음악만, 대화 없음)

  YOLO → label: "삼겹살" (파인튜닝 모델에 있다면)
  STT  → 매칭 없음 (대화 없음)
  CLIP → concept: "캠핑 야외 모닥불", ad_category: "운동_레저"

  Shopping_Ad 매칭:
    YOLO "삼겹살" + CLIP "캠핑" → 캠핑+고기 맥락
    → 홈쇼핑 "캠핑용 전기그릴 59,000원" 매칭
```

## 6. 알려진 한계

| 한계 | 영향 | 대응 |
|------|------|------|
| "김치", "소고기" 같은 반찬/식재료는 AI Hub에 단독 메뉴로 없음 | YOLO로 탐지 불가 | STT에서 키워드로 잡음 |
| 761종 중 하위 메뉴(음료류 3장)는 탐지 불가 | 해당 메뉴 광고 매칭 안 됨 | 실제 영향 미미 (음료 광고 수요 적음) |
| AI Hub 이미지 ↔ VOD 트레일러 도메인 갭 | 실제 VOD에서 성능 저하 가능 | A/B 테스트로 실측 후 판단 |
| CLIP 한국어 판별력 부족 (std=0.009) | 메뉴 구분 불가 | YOLO+STT가 메뉴 담당, CLIP은 장면만 |
| STT 키워드 156개 커버리지 한계 | 안 잡히는 키워드 존재 | 점진적 확충 |
| 말 안 하는 장면에서 STT 무력 | 시각 단독 판단 | YOLO 메뉴명 탐지로 보완 |
