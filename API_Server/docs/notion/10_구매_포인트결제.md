# POST /purchases — 포인트 구매

## 기능 개요

포인트를 차감하여 시리즈를 구매한다. purchase_history + point_history 동시 INSERT를 DB 트랜잭션으로 보장.

## 기본 기능

**요청**:
```json
POST /purchases
Authorization: Bearer {token}
{
  "series_nm": "눈물의 여왕",
  "option_type": "rental",
  "points_used": 490
}
```

**처리 로직**:
1. 포인트 잔액 확인: `point_history`에서 `SUM(CASE WHEN type='earn' THEN amount ELSE -amount END)`
2. 잔액 < points_used -> 402 에러
3. DB 트랜잭션 시작:
   - `purchase_history` INSERT (rental: `expires_at = NOW() + 48h`, permanent: `expires_at = NULL`)
   - `point_history` INSERT (`type='use'`, `related_purchase_id` 연결)
4. 트랜잭션 커밋

**응답**:
```json
{
  "purchase_id": 334001,
  "series_nm": "눈물의 여왕",
  "option_type": "rental",
  "points_used": 490,
  "remaining_points": 99510,
  "expires_at": "2026-03-22T15:30:00Z"
}
```

## 예외사항

| 에러 코드 | HTTP | 조건 | 메시지 |
|-----------|------|------|--------|
| `INVALID_OPTION_TYPE` | 400 | rental/permanent 아님 | 구매 유형은 rental 또는 permanent만 선택할 수 있습니다 |
| `INVALID_POINTS_AMOUNT` | 400 | points_used <= 0 | 포인트는 0보다 큰 값이어야 합니다 |
| `INSUFFICIENT_POINTS` | 402 | 잔액 부족 | 포인트가 부족합니다 (잔액: {balance}P, 필요: {required}P) |

## 제약사항

- 인증 필수
- 대여 만료: 48시간 확정
- 음수 잔액 방지: Frontend 팝업 + API 402 대응 (DB CHECK 제약 없음)
- 동일 시리즈 재구매 허용
- 포인트 잔액 = 실시간 집계 (캐시 없음, 향후 Redis 도입 검토)
