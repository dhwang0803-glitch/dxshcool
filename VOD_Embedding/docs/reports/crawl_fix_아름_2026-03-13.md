## 2. 원인 분석

### 핵심 버그: yt-dlp match_filter 거부 시 entry 반환

```python
# 기존 코드 (버그)
info = ydl.extract_info(f"ytsearch1:{query}", download=True)
if info and info.get('entries'):
    entry = info['entries'][0]
    filename = f"{entry['id']}.{entry.get('ext', 'webm')}"
    return {"status": "success", "filename": filename, ...}  # 파일 실제 존재 확인 없이 success 기록
```

yt-dlp `match_filter`(duration 30~300초)나 `max_filesize`(50MB)로 다운로드가 거부되어도
`extract_info`는 entry 정보를 반환함 → 코드가 파일 없이 success로 기록.

### phantom 분류
| 원인 | 건수 |
|------|------|
| duration > 300초 (풀에피소드 반환) | 4,079건 |

| 30~300초 정상 범위인데 파일 없음 (max_filesize 초과 추정) | 304건 |
| **합계** | **4,634건** |

## 3. 수정 내용

### 3-1. 핵심 버그 수정 (파일 존재 확인)

다운로드 후 파일 실제 존재 여부 확인, 없으면 다음 쿼리로 `continue`.

### 3-2. 설정값 조정

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| `DURATION_MAX_SEC` | 300초 | 600초 (예능 하이라이트 커버) |
| `MAX_FILESIZE_BYTES` | 50MB | 100MB |

### 3-3. dry_run 상태 분리

- 기존: `dry_run` → `"status": "success"` 기록 → 이후 실행에서 스킵됨 (버그)
- 수정: `"status": "dry_run"` → 상태 파일 미저장 → 재실행 가능

### 3-4. skip 조건 정리

`"success"` 만 스킵, `"skipped"` (dead branch) 제거.

### 3-5. 카운트 재계산 함수 도입

```python
def recompute_status_counts(status: dict, target_vod_ids: set = None):
    ...
```

누적 오차 방지, `save_status()` 호출 시 자동 재계산. `--status` 시 현재 대상 vod_id 집합 기준으로만 계산.

### 3-6. 2-step 다운로드 리팩터링

- 기존: `ytsearch1` + `download=True` → 1개 바로 다운
- 수정: `ytsearch3` + `download=False` → duration 조건 통과 후보 선택 → URL 1개만 다운로드
- 효과: 불필요한 다중 다운로드 제거, 조건 미통과 시 다음 쿼리로 자동 넘어감

### 3-7. 회차 정규식 확장

```python
# 수정 전
m = re.search(r'(\d+)\s*[회화]', asset_nm)

# 수정 후
m = re.search(r'(?:제\s*)?(\d+)\s*(?:회|화|부)\b', asset_nm)
# + EP.12, E12 패턴 추가
```

### 3-8. series명 추출 개선

위치 무관 전체 제거 → **끝부분만** 제거 (제목 중간 손상 방지).

```python
# 수정 전 (위치 무관)
name = re.sub(r'(?:제\s*)?\d+\s*(?:회|화|부)\b', '', asset_nm)

# 수정 후 (끝부분만)
name = re.sub(r'\s*(?:제\s*)?\d+\s*(?:회|화|부)\b\s*$', '', asset_nm)
```

### 3-9. 날짜 파싱 강화

```python
# 수정 전
date_str = re.sub(r'[^0-9]', '', release_date[:10])

# 수정 후 (전체 문자열, 앞 8자리 기준)
date_str = re.sub(r'[^0-9]', '', str(release_date))
if len(date_str) < 8:
    raise ValueError(...)
d = _date(int(date_str[:4]), int(date_str[4:6]), int(date_str[6:8]))
```

### 3-10. 기타 안정성 개선

- `DATA_DIR.mkdir()` → `logging.basicConfig()` 이전으로 이동 (로그 파일 생성 실패 방지)
- `sys.stdout.reconfigure` → `try/except` 감싸기
- `duration=None` 항목 → fallback 후보로 보존 (조건 통과 후보 없을 때 시도)
- `match_filter` → `ydl_opts_dl`에 복구 (fallback 항목 다운로드 안전망)
- `error` 상태 → `failed` 통계에 포함
- `outtmpl` → `{vod_id}__%(id)s.%(ext)s` (vod_id 추적성 확보)
- 완료 로그 → 이번 실행 기준 (`run_success`, `run_failed`, `run_skipped`)
- FAIL 로그 → `사유={reason}` 추가
