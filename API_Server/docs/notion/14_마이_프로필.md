# GET /user/me/profile — 유저 프로필

## 기능 개요

유저의 표시명과 포인트 잔액을 반환한다.

## 기본 기능

**요청**:
```
GET /user/me/profile
Authorization: Bearer {token}
```

**처리 로직**:
1. `public."user"` 에서 sha2_hash 존재 확인
2. `user_name` = sha2_hash 앞 5자 (예: `a1b2c`)
3. `point_balance` = `point_history` 실시간 집계:
   ```sql
   SELECT COALESCE(
     SUM(CASE WHEN type = 'earn' THEN amount ELSE -amount END), 0
   ) FROM point_history WHERE user_id_fk = $1
   ```

**응답**:
```json
{
  "user_name": "a1b2c",
  "point_balance": 100000
}
```

## 예외사항

| 에러 코드 | HTTP | 조건 | 메시지 |
|-----------|------|------|--------|
| `PROFILE_NOT_FOUND` | 404 | 유저 없음 | 사용자 정보를 찾을 수 없습니다 |

## 제약사항

- 인증 필수
- **쿠폰 기능 제거 확정** (coupon_count 필드 없음)
- point_balance는 매 요청 시 실시간 집계 (캐시 없음, 향후 Redis 도입 검토)
- 비식별 데이터 — 실명 없이 sha2_hash 앞 5자로 표시

## 업스트림 의존성

| 테이블 | 컬럼 | 타입 | 용도 |
|--------|------|------|------|
| `public."user"` | `sha2_hash` | VARCHAR | 존재 확인 + 표시명 파싱 |
| `public.point_history` | `user_id_fk`, `type`, `amount` | VARCHAR/INTEGER | 잔액 집계 |
