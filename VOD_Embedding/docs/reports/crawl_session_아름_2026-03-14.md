## 2. 발견된 문제 및 수정 내역

### 2-1. 핵심 수정 사항

| 문제 | 원인 | 수정 |
|------|------|------|
| 관련 클립 오매칭 | `series_words[:2]` ANY 매칭 → 부분 일치로 엉뚱한 영상 통과 | `series_key` 전체 문자열 매칭으로 변경 |
| 시즌 표기 불일치 | DB "위대한 탄생 시즌2" ≠ YouTube "위대한탄생2" | `series_key_alt = re.sub(r'시즌\s*(\d+)', r'\1', series_key)` 추가 |
| 시즌 쿼리 미매칭 | 검색어도 "시즌2" 포함 → YouTube 검색 결과 누락 | `series_short` 쿼리 추가 (e.g. "위대한 탄생2 E01") |
| fallback 오매칭 | `title_filtered` 비면 `valid_entries` 전체 허용 → 뉴스/클립 저장 | `title_filtered` 비면 즉시 `continue` (fallback 차단) |
| 회차 오매칭 | 시리즈명만 확인, 회차 번호 미검증 → 다른 회차 영상 저장 | `ep_int` 회차 검증 추가 (E01/1회/#1 등) |
| 중복 yt_id | 같은 파일이 여러 회차에 저장됨 | `used_yt_ids` 세트로 중복 차단 |
| DNS 오류 장기 대기 | `ydl_opts_info`에 retries 없음 → 기본 10회 재시도 (2~4시간 멈춤) | `retries:2`, `extractor_retries:0` 추가 |

### 2-2. 최종 필터 구조 (crawl_trailers_아름.py)

```
[1] title_matches: series_key 또는 series_key_alt 포함
    → 없으면 continue (fail)
[2] ep_filtered: ep_int 있으면 제목에 회차번호 (E01/1회/#1 등) 포함 필수
    → 없으면 continue (fail)
[3] 중복 yt_id 차단: used_yt_ids 세트로 이미 저장된 영상 재사용 방지
[4] duration 필터: 30~600초
[5] 다운로드 → 파일 존재 확인 → success
```

## 3. 수집 전략 전환 (정확도 우선)

### 배경

기존 "커버리지 우선" 전략으로 돌렸을 때 실제 오매칭 사례:
- `위대한탄생 시즌2 02회` → 시험관 시술 출연자 사연 클립 저장
- `위대한탄생 시즌2 05회` → Star Audition #07 (다른 회차) 저장
- `나는가수다2 09회`, `11회` → 동일 예고편 파일 중복 저장

### 전환 방향

**정확도 우선 + 실패 항목 2차 재시도** 구조로 변경.

```
1차 수집 (현재)
  - 조건: 시리즈명 일치 + 회차 번호 일치 + 중복 yt_id 없음
  - 애매하면 전부 failed 처리
  - 영어 제목 영상도 이 단계에서는 과감히 포기

2차 재시도 (failed 항목만)
  - 대상: 1차 failed 항목
  - 영어 쿼리 추가 (예: "I Am a Singer2 #08", "Star Audition #07")
  - 방송사 채널 기준 쿼리 추가

3차 수동 검토
  - 2차에서도 실패한 항목 → 유튜브 원본 부재 가능성으로 분류
```

### 영어 제목 문제 (2차 과제)

MBCkpop 등 방송사 공식 채널이 영어 제목으로 업로드하는 경우:
- 예: `나는가수다2 08회` → YouTube에는 `"#08, I Am a Singer2 20120812"` 로 업로드
- 1차에서는 `series_key = "나는가수다2"` 가 영어 제목에 없어서 failed 처리
- **2차 재시도 때 영어 쿼리로 보완** 예정

영어 제목 매핑을 1차 파이프라인에 억지로 넣으면 예외처리 복잡도가 급격히 증가하므로,
2차 재시도 단계에서만 적용하는 것이 실무적으로 맞는 방향.
