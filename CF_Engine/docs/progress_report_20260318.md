## 4. Step 3 — 추천 점수 급락 지점 분석 (2026-03-18 확정)

5,000명 샘플 기준 유저별 ALS 추천 점수 급락 지점을 분석하여 최적 K 설계 근거를 확보했습니다.

### 결과 (필터 OFF 기준)

| 기준 | K | 비고 |
|------|---|------|
| 50% 유저 커버 | K=1 | 절반 이상이 1위에서 급락 |
| 75% 유저 커버 | **K=2** | 서비스 최소 의미 있는 추천 |
| 90% 유저 커버 | K=4 | - |
| 현행 설정 | **K=10** | 다양성 목적 유지 |

- **65.3%의 유저가 1위에서 바로 급락** — 시청 집중도(특정 시리즈 편중) 패턴 때문

### cutoff=1 유저 파일럿 검증 (10명)

cutoff=1 유저 10명 샘플에서 추천 품질 점수화:

| 등급 | 유저 수 | 비율 |
|------|---------|------|
| ★★★★★ 매우 좋음 | 1명 | 10% |
| ★★★★☆ 좋음 | 8명 | 80% |
| ★★★☆☆ 보통 | 1명 | 10% |

- 평균 장르 적합율: **91.0%** → 급락이 있어도 추천 자체의 장르 연관성은 높음
- 1위 이후 추천의 장르 연관성은 유지되나 점수 하락 구간 확인

## 5. Step 5 — 감독/배우 선호 기반 추천 후처리 (2026-03-18 구현 완료)

### 설계 방침

기존 ALS 추천 결과 뒤에 **최대 2개**를 선택적으로 추가하는 방식으로,
DB 스키마 변경 없이 `recommendation_type: COLLABORATIVE` 그대로 사용합니다.

### 동작 방식

```
ALS Top-20 추천 생성
    ↓
유저 시청 이력 분석
    → 동일 감독 3편 이상? → 해당 감독 미시청 VOD 1개 추가 (rank 21)
    → 동일 배우 3편 이상? → 해당 배우 미시청 VOD 1개 추가 (rank 22)
    ↓
후보 조건: 미시청 + 품질 필터(poster+embedding) 통과 VOD
```

- cast_lead / cast_guest 컬럼이 JSON 배열 `["현빈", ...]`과 쉼표 구분 `"정형돈, 데프콘"` **혼재** → `parse_cast()` 함수로 양쪽 처리
