# WS /ad/popup — 광고 팝업 (WebSocket)

> 구현 예정 — Shopping_Ad 브랜치 연동 후 착수

## 기능 개요

VOD 재생 중 사물인식 트리거 기반 광고 팝업을 실시간 WebSocket으로 전송한다.
지역광고(축제)와 제철장터(식재료) 2가지 타입.

## 기본 기능

### 타입 1: 지역광고 (`local_gov`)

- 지역 사진 + 축제 이름 + 축제 일정이 포함된 움직이는 광고 팝업 출력
- 유저 선택 버튼 없음
- **10초 후 자동 최소화** -> 파란색(다시 열기) / 빨간색(완전 제거) 작은 버튼만 남김

**Server -> Client**:
```json
{
  "type": "ad_popup",
  "ad_type": "local_gov",
  "vod_id": "cjc|M1234...",
  "time_sec": 120,
  "data": {
    "image_url": "https://example.com/festival.jpg",
    "festival_name": "보령머드축제",
    "festival_schedule": "2026-07-18 ~ 2026-07-27"
  }
}
```

### 타입 2: 제철장터 (`seasonal_market`)

- "채널 25번 제철장터에서 현재 {사물인식된 상품명}이 판매중입니다" 메시지
- 버튼: **시청예약하기** / **종료하기**
- 시청예약 클릭 -> "시청예약되었습니다" 팝업 2초 후 자동 제거
- 종료하기 클릭 -> 즉시 팝업 제거
- **10초 후 미응답 시 자동 최소화** -> 파란색/빨간색 버튼

**Server -> Client**:
```json
{
  "type": "ad_popup",
  "ad_type": "seasonal_market",
  "vod_id": "cjc|M1234...",
  "time_sec": 120,
  "data": {
    "product_name": "제주 감귤",
    "channel": 25,
    "broadcast_time": "14:00"
  }
}
```

### Client -> Server 액션

```json
{ "type": "ad_action", "action": "reserve_watch", "vod_id": "..." }
{ "type": "ad_action", "action": "dismiss", "vod_id": "..." }
{ "type": "ad_action", "action": "minimize", "vod_id": "..." }
{ "type": "ad_action", "action": "reopen", "vod_id": "..." }
```

| action | 설명 | 서버 응답 |
|--------|------|----------|
| `reserve_watch` | 시청 예약 | `{"type": "ad_response", "result": "reserved"}` -> 2초 후 팝업 제거 |
| `dismiss` | 완전 제거 | 팝업 즉시 제거 |
| `minimize` | 최소화 | 파란/빨간 버튼만 표시 |
| `reopen` | 다시 열기 | 원래 팝업 복원 |

## 예외사항

- WebSocket 연결 끊김 시 광고 상태 소실 (재연결 시 복원 여부 미정)
- 해당 VOD에 트리거 포인트 없으면 팝업 미발생

## 제약사항

- 인증 필수 (JWT 토큰으로 WebSocket 핸드셰이크)
- 양방향 WebSocket (팝업 축소/확대/제거 인터랙션 필요)
- 채널이동 기능 제거 (시청예약만 유지)
- Shopping_Ad 브랜치의 `serving.shopping_ad` 테이블 데이터 필요

## 업스트림 의존성

| 테이블 | 컬럼 | 타입 | 용도 |
|--------|------|------|------|
| `serving.shopping_ad` | `vod_id_fk`, `ts_start`, `ts_end`, `ad_category`, `score`, `ad_hints`, `product_name`, `image_url`, `channel` | 각종 | 트리거 포인트 + 광고 데이터 |
| `public.tv_schedule` | `channel`, `broadcast_date`, `start_time`, `program_name` | 각종 | 제철장터 방송 시간 |

## 다운스트림 의존성

| 대상 | 용도 |
|------|------|
| 시청 예약 저장 (미설계) | `reserve_watch` 액션 시 예약 기록 저장 필요 |
