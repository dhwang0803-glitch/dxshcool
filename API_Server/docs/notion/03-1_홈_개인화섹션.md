# GET /home/sections/{user_id} — 개인화 섹션

## 기능 개요

유저의 장르별 시청 비중 내림차순으로 VOD 섹션을 구성한다. 시청 이력이 없으면 CT_CL 4종 고정 응답으로 fallback.

## 기본 기능

**요청**:
```
GET /home/sections/{user_id}
Authorization: Bearer {token}
```

**처리 로직**:
1. `home_service.get_personalized_sections(user_id)` 호출
2. 시청 이력 있음 → 장르별 시청 비중 내림차순 + 미시청 장르 도전 구성
3. 시청 이력 없음 → CT_CL 4종 고정 섹션 fallback (`/home/sections`와 동일 데이터)

**응답** (개인화):
```json
{
  "sections": [
    {
      "genre": "드라마",
      "view_ratio": 45,
      "vod_list": [
        {
          "series_nm": "눈물의 여왕",
          "asset_nm": "눈물의 여왕 01회",
          "poster_url": "/posters/..."
        }
      ]
    },
    {
      "genre": "액션",
      "view_ratio": 0,
      "vod_list": [...]
    }
  ]
}
```

**응답** (fallback — 시청 이력 없음):
```json
{
  "sections": [
    {
      "genre": "TV 드라마",
      "view_ratio": 0,
      "vod_list": [
        {
          "series_nm": "눈물의 여왕",
          "asset_nm": "눈물의 여왕 01회",
          "poster_url": "/posters/..."
        }
      ]
    }
  ]
}
```

- `view_ratio`: 시청 비중 (0~100 정수). fallback 시 모두 0
- `vod_list`: `series_nm`, `asset_nm`, `poster_url` 3개 필드

## 예외사항

- 시청 이력 없으면 fallback (에러 아님)

## 제약사항

- 인증 필수
- 기존 `/home/sections`(비로그인/신규 유저)와 분리된 엔드포인트

## 업스트림 의존성

| 테이블 | 컬럼 | 타입 | 용도 |
|--------|------|------|------|
| 시청 이력 (집계) | `user_id`, `genre`, `view_ratio` | 각종 | 장르별 시청 비중 |
| `serving.popular_recommendation` | `vod_id_fk`, `ct_cl` | 각종 | fallback용 CT_CL 4종 |
| `public.vod` | `series_nm`, `asset_nm`, `poster_url` | VARCHAR/TEXT | VOD 메타데이터 |
