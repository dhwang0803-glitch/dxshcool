# WS /ad/popup — 광고 팝업 (WebSocket)

> `serving.shopping_ad` DB 조회 + 시청예약 연동 구현 완료

## 기능 개요

VOD 재생 중 타임스탬프 기반 광고 팝업을 실시간 WebSocket으로 전송한다.
지역광고(축제)와 제철장터(식재료) 2가지 타입.

## 현재 구현 상태

- WebSocket 연결/해제/메시지 수신 **구현 완료**
- 클라이언트 연결 관리 (`user_id → WebSocket` 매핑)
- 유저별 활성 광고 상태 관리 (재연결 시 미제거 광고 복원 전송)
- **`playback_update` 수신 → `serving.shopping_ad` DB 조회 → 광고 푸시** 구현 완료
- **중복 전송 방지** (`_sent_ad_ids`로 같은 광고 재전송 차단)
- 액션 처리: `reserve_watch` (**`watch_reservation` DB INSERT 연동 완료**), `dismiss` (활성 광고 제거), `minimize`/`reopen` (클라이언트 UI)
- 외부 모듈 호출용 `send_ad_to_user()` 함수 제공

## 기본 기능

### 타입 1: 지역광고 (`local_gov`)

- 지역 사진 + 축제 이름 + 축제 일정이 포함된 움직이는 광고 팝업 출력
- 유저 선택 버튼 없음
- **10초 후 자동 최소화** -> 파란색(다시 열기) / 빨간색(완전 제거) 작은 버튼만 남김

**Server -> Client**:
```json
{
  "type": "ad_popup",
  "ad_type": "local_gov",
  "vod_id": "cjc|M1234...",
  "time_sec": 120,
  "data": {
    "image_url": "https://example.com/festival.jpg",
    "festival_name": "보령머드축제",
    "festival_schedule": "2026-07-18 ~ 2026-07-27"
  }
}
```

### 타입 2: 제철장터 (`seasonal_market`)

- "채널 25번 제철장터에서 현재 {사물인식된 상품명}이 판매중입니다" 메시지
- 버튼: **시청예약하기** / **종료하기**
- 시청예약 클릭 -> "시청예약되었습니다" 팝업 2초 후 자동 제거
- 종료하기 클릭 -> 즉시 팝업 제거
- **10초 후 미응답 시 자동 최소화** -> 파란색/빨간색 버튼

**Server -> Client**:
```json
{
  "type": "ad_popup",
  "ad_type": "seasonal_market",
  "vod_id": "cjc|M1234...",
  "time_sec": 120,
  "data": {
    "product_name": "제주 감귤",
    "channel": 25,
    "broadcast_time": "14:00"
  }
}
```

### Client -> Server: 재생 시간 업데이트

프론트엔드가 VOD 재생 중 주기적으로 전송한다. 서버가 해당 타임스탬프에 매칭되는 광고를 DB에서 조회하여 푸시한다.

```json
{ "type": "playback_update", "vod_id": "cjc|M1234...", "time_sec": 120.5 }
```

### Client -> Server: 광고 액션

```json
{ "type": "ad_action", "action": "reserve_watch", "vod_id": "..." }
{ "type": "ad_action", "action": "dismiss", "vod_id": "..." }
{ "type": "ad_action", "action": "minimize", "vod_id": "..." }
{ "type": "ad_action", "action": "reopen", "vod_id": "..." }
```

| action | 설명 | 서버 응답 |
|--------|------|----------|
| `reserve_watch` | 시청예약 (제철장터 ch.25 → `watch_reservation` INSERT) | 성공: `{"type": "ad_response", "action": "reserve_watch", "message": "시청예약되었습니다"}` / 실패: `{"error": "시청예약에 실패했습니다"}` |
| `dismiss` | 완전 제거 | 활성 광고 목록에서 제거 (재연결 시 복원 방지) |
| `minimize` | 최소화 | 파란/빨간 버튼만 표시 (클라이언트 UI) |
| `reopen` | 다시 열기 | 원래 팝업 복원 (클라이언트 UI) |

## WebSocket 연결

```
ws://host/ad/popup?user_id={sha2_hash}
```

- `user_id` query param 필수 (없으면 `4001` 코드로 연결 거부)
- 재연결 시 미제거(dismiss 안 된) 광고 자동 복원 전송

## 예외사항

- WebSocket 연결 끊김 시 광고 상태 **메모리 보존** (재연결 시 복원)
- `user_id` 없이 연결 시 즉시 `close(4001)`
- 잘못된 JSON → `{"error": "invalid JSON"}` 전송
- 알 수 없는 메시지 타입 → `{"error": "unknown message type: xxx"}` 전송
- 해당 VOD에 트리거 포인트 없으면 팝업 미발생

## 제약사항

- 인증: query param `user_id` (JWT 핸드셰이크 아님)
- 양방향 WebSocket (팝업 축소/확대/제거 인터랙션 필요)
- 채널이동 기능 제거 (시청예약만 유지)
- Shopping_Ad 브랜치의 `serving.shopping_ad` 테이블 데이터 필요

## 업스트림 의존성

| 테이블 | 컬럼 | 타입 | 용도 |
|--------|------|------|------|
| `serving.shopping_ad` | `shopping_ad_id`, `vod_id_fk`, `ts_start`, `ts_end`, `ad_category`, `signal_source`, `score`, `ad_hints`, `ad_action_type`, `ad_image_url`, `product_name`, `channel`, `expires_at` | 각종 | 타임스탬프 기반 광고 조회 (WHERE ts_start <= $t AND ts_end >= $t AND expires_at > NOW()) |
