# POST /series/{series_nm}/episodes/{asset_nm}/progress — 진행률 기록

## 기능 개요

에피소드 시청 진행률을 기록한다. 기존 기록이 있으면 UPDATE, 없으면 INSERT (UPSERT).

## 기본 기능

**요청**:
```json
POST /series/{series_nm}/episodes/{asset_nm}/progress
Authorization: Bearer {token}
{ "completion_rate": 75 }
```

**처리 로직**:
1. `asset_nm` -> `full_asset_id` 변환 (vod 테이블에서 `series_nm + asset_nm` 조회)
2. `episode_progress` 테이블에 UPSERT:
   ```sql
   INSERT INTO episode_progress (user_id_fk, vod_id_fk, series_nm, completion_rate, watched_at)
   VALUES ($1, $2, $3, $4, NOW())
   ON CONFLICT (user_id_fk, vod_id_fk)
   DO UPDATE SET completion_rate = $4, watched_at = NOW()
   ```

**응답**:
```json
{
  "episode_title": "눈물의 여왕 05회",
  "completion_rate": 75,
  "watched_at": "2026-03-20T15:30:00Z"
}
```

## 예외사항

| 에러 코드 | HTTP | 조건 | 메시지 |
|-----------|------|------|--------|
| `INVALID_COMPLETION_RATE` | 400 | completion_rate가 0~100 범위 밖 | 시청 진행률은 0~100 범위여야 합니다 |
| `EPISODE_NOT_FOUND` | 404 | series_nm + asset_nm 조합이 vod에 없음 | 해당 에피소드를 찾을 수 없습니다 |

## 제약사항

- 인증 필수
- `completion_rate`: SMALLINT 0~100 정수
- 업데이트 주기: **재생 종료 시 1회** (향후 Redis 실시간 수신 -> 매일 자정 DB 반영 전환 예정)
- PK = `(user_id_fk, vod_id_fk)` — 동일 유저+에피소드는 항상 1행

## 업스트림 의존성

| 테이블 | 컬럼 | 타입 | 용도 |
|--------|------|------|------|
| `public.vod` | `full_asset_id`, `series_nm`, `asset_nm` | VARCHAR | asset_nm -> full_asset_id 매핑 |

## 다운스트림 의존성

| 테이블 | 컬럼 | 타입 | 용도 |
|--------|------|------|------|
| `public.episode_progress` | `user_id_fk`, `vod_id_fk`, `series_nm`, `completion_rate`, `watched_at` | 각종 | UPSERT |
