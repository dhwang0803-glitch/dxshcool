## 재작업 배경

### 문제 1 — vod_meta_embedding: 시리즈 그룹핑으로 회차별 메타데이터 미반영

기존 `run_meta_embed_parquet.py` (v1)는 시리즈 연산량 절감을 위해 **시리즈 단위 그룹핑 → 대표 1개 인코딩 → 전체 복사** 방식을 사용했다.

```
런닝맨 350회 → 런닝맨 대표 row의 smry/cast_guest 벡터 복사
런닝맨 351회 → 동일 벡터 (회차별 게스트/줄거리 무관)
...
166,159건 → 23,541개 대표 벡터만 생성 후 복사
```

TV 연예/오락(예능)은 회차마다 출연 게스트, 주제, 장소가 다르다.
특정 게스트의 팬인 사용자에게 해당 게스트가 출연한 회차를 추천하려면
**회차별 cast_guest, smry 가 각각의 임베딩 벡터에 반영**되어야 한다.
그러나 v1 방식은 대표 row의 메타데이터만 사용했으므로 개인화 추천 품질이 저하된다.

### 문제 2 — vod_embedding: 에피소드 공유 파일로 동일 벡터 복사

기존 `batch_embed.py`는 `file_groups = defaultdict(list)`를 사용하여
**같은 트레일러 파일을 공유하는 vod_id들에 동일 벡터를 복사**했다.

```
9,570건 → 829개 고유 파일 → 829개 벡터 생성 → 나머지 8,741건 복사
```

근본 원인은 `crawl_trailers.py`의 검색 쿼리가 예능도 **시리즈명 기준**으로 작성되어 있었기 때문이다.

```
런닝맨 350회 → 쿼리: "런닝맨 예고편" → 동일 파일 다운로드
런닝맨 351회 → 쿼리: "런닝맨 예고편" → 동일 파일 다운로드
```

## 재작업 범위 및 수정 내용

### 1. `run_meta_embed_parquet.py` v1 → v2

| 항목 | v1 (기존) | v2 (수정) |
|------|-----------|-----------|
| 처리 단위 | 시리즈 대표 1개 | 에피소드 개별 |
| 인코딩 건수 | 23,541건 | 166,159건 |
| cast_guest 반영 | 대표 row 기준 | 해당 회차 기준 |
| smry 반영 | 대표 row 기준 | 해당 회차 기준 |
| 체크포인트 키 | `done_series_keys` | `done_vod_ids` |
| 체크포인트 파일 | `meta_embed_checkpoint.json` | `meta_embed_checkpoint_v2.json` |
| 체크포인트 주기 | 20 시리즈 | 20 VOD |

**핵심 변경**: `group_by_series()` / `pick_representative()` 제거,
`pending = [v for v in all_vods if v["full_asset_id"] not in done_ids]` 로 전체 에피소드 flat 처리.

### 2. `crawl_trailers_아름.py` (신규)

기존 `crawl_trailers.py` 대신 박아름 담당 TV 연예/오락 앞 절반을 **에피소드별 개별 쿼리**로 재수집.

| 항목 | 기존 | 신규 |
|------|------|------|
| 검색 쿼리 | `{시리즈명} 예고편` (시리즈 고정) | 에피소드별 우선순위 쿼리 |
| 트레일러 저장 경로 | `data/trailers/` | `data/trailers_아름/` |
| 상태 파일 | `crawl_status.json` | `crawl_status_아름.json` |
| DB 조회 필드 | tasks_A.json (release_date 없음) | DB 직접 조회 (release_date 포함) |

**에피소드별 검색 쿼리 우선순위:**
1. `{시리즈명} {N}회` — asset_nm에서 회차 번호 추출
2. `{시리즈명} {YYYY}년 {M}월 {D}일` — DB release_date 활용
3. `{시리즈명} 예고편` — 시리즈 fallback
4. `{시리즈명} 하이라이트`
5. `{시리즈명} 1회`

### 3. `batch_embed_아름.py` (신규)

기존 `batch_embed.py` 대신 에피소드별 개별 CLIP 임베딩.

| 항목 | 기존 | 신규 |
|------|------|------|
| 파일 그룹핑 | `file_groups` defaultdict (동일 파일 → 복사) | 제거 (각 vod_id 독립 처리) |
| 크롤링 소스 | `crawl_status.json` | `crawl_status_아름.json` |
| 트레일러 경로 | `data/trailers/` | `data/trailers_아름/` |
| 상태 파일 | `embed_status.json` | `embed_status_아름.json` |
| 출력 파일 | `embeddings_아름.parquet` | `embeddings_아름_v2.parquet` |
| 체크포인트 주기 | 50건 | 50건 (동일) |
